# Mercury AI Diligence — Template / Structural Change Requests

> Purpose: Capture **structure/template changes** (not specific content edits) to apply across the deck.

---

## 1) Executive Summary design (Page 2 template)
- Page 2 should be a **true Executive Summary**, not a “chapter list.”
- Under each section title, include **1–2 executive-summary bullets** only.
- Create a consistent mechanism so these bullets are **pulled from each section’s own executive summary** (1:1 mapping).

## 2) “Gaps” handling (global template rule)
- Remove “Gaps” callouts from individual section slides.
- Create **one to a few consolidated “Gaps / Open Questions” slides at the end**, organized by section (Growth Thesis, Disruption, Asset Value, AI Assessment, Synergies, etc.).
- For pages where “no evidence/public evidence” language is used, standardize on **“Not yet demonstrated”** where applicable.

## 3) Add internal value and quantification pages (after Page 26)
- Reframe Page 26 as **External Value Examples** (product-driven).
- Add Page 26A:
  - **Internal Value Proofs (Cost, Speed, Productivity)** tied to CI/CD and AI-assisted engineering.
- Add Page 26B:
  - **Quantification placeholder** for later valuation/impact estimates.

## 4) Architecture slide format (Page 24 template)
- Remove low-signal vendor/tooling callout boxes listing components (e.g., Vue.js/CloudFront/Cognito).
- Add standardized qualitative assessment boxes (matching Page 20 style):
  - **Strong**
  - **Adequate for stage**
  - **Gap for scale**
- Require 1–3 bullets per box.

## 5) Development / AI engineering practices simplification (Page 21 template)
- Consolidate “development practices” + “AI-assisted engineering” into one row.
- Remove template rows:
  - “customer-facing AI deployment”
  - standalone “continuous AI operating model”
- Fold operating model content into **AI governance & ownership**.

## 6) Relationship asset page taxonomy (Page 17 template)
- Standardize relationship categories to:
  - **Sites**
  - **Sponsors**
  - **Channel partners**
- Remove advisory/financial relationship categories from the template.

## 7) Underlying asset value (Harvey ball) style rules (Page 13 template)
- Remove numeric annotations on Harvey balls (e.g., “75”, “50”) — keep purely visual.
- Standardize scoring granularity; avoid mixed quantitative/qualitative signals.

## 8) Data asset strength scoring colors (Page 14 template)
- Replication column color rules:
  - “High” should be consistently positive (green), not red.
- Reduce scoring gradients (e.g., eliminate “very high” if not meaningful).

## 9) Synergy pathways slide redesign (Page 30 template)
- Replace networked “do everything at once” diagram with staged plan:
  - **Wave 1 / Wave 2 / Wave 3**
- Each wave should include:
  - 2–4 bullets max
  - clear sequencing rationale
  - explicit partner timing (e.g., Florence wave 2; Clean Spirit investigation wave 1 then integration later)

## 10) Remove redundant slides
- Remove Page 28 entirely.

## 11) Remove low-signal boxes that clutter
- Remove “resolved questions” / “results questions” template boxes where they don’t add diligence signal (e.g., Team/Operating Model, AI Assessment).
